二叉树

二叉树：
根
父
子
左
右

1.良好查询性能
2.有很好的新增和删除的性能
3.若左子树不空，则右子树上所有节点的值均小于它根节点的值
4.若右子树不空，则右子树上所有节点的值均大于它根节点的值
5.左，右子树也分别为二叉排序树

二叉树，双向链表：如何区分
双向链表的地址一个上一个元素的地址，一个下一个元素的地址
二叉树中的两个地址，都是下一个元素的地址

前序遍历：根左右
—-中序遍历：左根右
后序遍历：左右根

删除
情况一：叶子节点
1.删除改节点
2.将父节点（左或右）指针置NULL
情况二：只有一个子树
1.删除该节点
2.将父节点（左或者右）指针指向子树
情况三：左右子树都有
1.用右子树最小的节点取代源节点 //左找最大替换
2.再递归删除最小节点

引用类型

能组成指针的就可以使用引用：
C++里边的
引用只能赋值一次，不能重新赋值
对象别名
（引用可以理解成是编译器维护的一个指针，但不占用空间）（海哥）
引用可以像指针那样去访问，修改对象的内容，但更加安全

引用
void test(int& x)
{
x = 1;
}
int main()
{
int a = 2;
test(a);
}

指针
void test(int* x)
{
*x = 1;
}
int main()
{
int a = 2;
test(&a);
}
俩都能对x进行赋值
引用跟指针在底层上编译完以后看不出任何区别
引用编译的时候就是指针，但编译器不允许你这个引用再指向别的地方

const 只能读，不能改

一个函数想访问私有成员的时候，正常情况下是不允许访问的
访问办法：
把这个函数，声明成那个类的朋友
即：友元函数 （交朋友）
运算符重载的时候，可能用友元
两个类共享数据
—友元函数与类的成员函数区别
成员函数有this指针，而友元函数没有this指针
友元函数是不能被继承的

call		nh_malloc_dbg		_heap_alloc_dbg	_heap_alloc_base	HeapAlloc

当有多个call的时候，怎么判断哪个call做分配内存的工作
把当前的指针拖到（栈）里，malloc进行完，它肯定要得到一个指针的指向，—即刚刚分配好的内存，
把得到的指向输入，得到这个内存的地址，进一步的call看看这个call干了什么，从而找到我们需要的那个call。
